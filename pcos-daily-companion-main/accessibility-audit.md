# 暖巢可访问性核验记录

本次核验覆盖首页、打卡、队伍、论坛、成长、洞察、健康日历和个人中心八个核心页面。审查重点包括原生控件语义、键盘可达性、可见焦点、移动端触控区域，以及侧边栏和账户菜单的辅助技术支持。该记录反映代码与浏览器预览审查结果；正式上线后仍建议结合真实设备和读屏软件进行用户验收。

| 核验维度 | 结果 | 证据与处理 |
|---|---|---|
| 键盘焦点 | 通过 | 全局样式为 `:focus-visible` 添加 3px 暖色描边与外扩光晕；同时尊重 `prefers-reduced-motion`。 |
| 首页 | 通过 | 英雄区、开始打卡、队伍状态与任务入口均使用原生 `button` 或组件按钮；团团插画含读屏标签。 |
| 打卡页 | 通过 | 打卡任务、经期操作、编辑弹窗和症状贴纸均为原生交互控件；症状贴纸与数值选择组均提供 `aria-pressed`。 |
| 队伍页 | 通过 | 创建、邀请及跳转操作均为有文本标签的标准按钮；空状态不伪造成员数据。 |
| 论坛页 | 通过 | 发布入口、草稿操作与话题入口均使用标准按钮及文本域；不预置虚构用户帖子。 |
| 成长页 | 通过 | 花园与团团插画提供描述性标签，主要操作提供清晰文字按钮。 |
| 洞察页 | 通过 | 报告生成操作使用标准按钮，自动周报开关采用语义化开关组件。 |
| 健康日历 | 通过 | 月份切换按钮有明确中文 `aria-label`；日期格为原生按钮，带日期标签与 `aria-pressed` 选中状态。 |
| 个人中心 | 通过 | 档案字段使用带标签的标准输入控件，保存操作使用明确文本按钮。 |
| 移动端触控 | 通过 | 在 390 × 844 预览下逐页检查；顶栏、主要行动按钮、任务卡、日历日期格与症状贴纸均保持独立、可见的触控区域。 |
| 侧边栏 | 通过 | 导航容器使用语义化 `ul`/`li`/`button`；导航项具备原生键盘可达性和焦点环，移动端采用可聚焦的抽屉组件。 |
| 账户下拉菜单 | 通过 | 基于 Radix Dropdown Menu 原语，触发器与菜单项保留键盘导航、焦点管理和展开状态支持。 |

> 本应用提供日常健康记录与陪伴性洞察，不替代医疗诊断、治疗或紧急医疗帮助。
